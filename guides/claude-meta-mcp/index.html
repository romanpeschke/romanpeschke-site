How to Connect Claude to Meta Ads Without Getting Banned

The official MCP, the new CLI, the raw API, and how to pick the right one for your account.

Tools covered
Claude Meta Ads MCP Meta Ads CLI

The official Meta Ads MCP server and CLI are here. That means you can fully manage your accounts with AI without getting banned. Below is the connect, plus how to pick MCP vs CLI vs raw API for your situation.

Pick your path
1

Working in Claude Desktop or chat? Use the MCP.

Fastest setup. No developer app. Best for analysis, reporting, and supervised changes. The MCP also exposes a few endpoints the CLI does not, like Signal Diagnostics and Help Center search.

2

Working in Claude Code with scripted reporting and QA? Use the CLI.

Same tool coverage as the MCP. Runs in your terminal, in cron, in CI/CD. Use this when you want repeatable account ops without building the full API stack.

3

Need CAPI, offline conversions, webhooks, or local targeting? Use the raw Marketing API.

The MCP and CLI are a curated tool set. Anything outside it (Conversions API server events, offline conversion uploads, webhooks, batch jobs, local geo targeting) needs the raw API. Highest setup burden, full surface, full control.

4

Need IG DMs, organic IG publishing, FB Page posts, or WhatsApp?

Different APIs entirely. None of these are in the Meta Ads MCP or CLI. They each have separate API surfaces, permissions, and usually a separate developer app.

Connect the MCP in 60 seconds
5

Open Claude and add a custom connector

Click + in any chat, or go to Customize → Connectors. Then click Add custom connector. Free Claude users only get one custom connector total, so make sure your slot is open.

6

Paste the Meta MCP URL

Paste https://mcp.facebook.com/ads and click Add.

7

Sign into Meta and pick your ad account

Authorize through Business OAuth. Pick the ad account you want connected. Read the requested permissions before you approve. You can revoke access in your Meta Business settings any time.

8

Toggle the connector on in your chat

Open the + menu in your chat, find Meta under Connectors, and toggle it on for that conversation.

9

Confirm anything you create stays paused

Anything new you build through the MCP comes through paused. Confirm it in Ads Manager before flipping anything live.

First prompts to try
10

Pull last 7 days of spend, CTR, and CPA by ad set.

Confirms the connection works and gives you a clean baseline read.

11

Find my worst-performing ad set in the last 14 days and tell me what to do.

Tests the model's ability to reason on your account, not just report numbers back.

12

Draft a new campaign targeting [audience] with a $50/day budget. Don't launch it.

Tests the write path. The MCP creates everything paused, so this is safe to run without risk.

Local business warning

Do not rely on country-only targeting for live local-business launches.

The underlying Marketing API supports city, ZIP, radius, and geo exclusions. Current public CLI docs show country-level targeting for ad set creation. If you run a restaurant, dentist, gym, home service, real estate, or multi-location franchise account: use the CLI for reports and QA, draft paused only, then patch targeting in Ads Manager or use the raw API before going live.

Watch out for

X
Free Claude plans cap you at one custom connector total. If you have already used your slot for something else, the Meta MCP will not fit until you remove the existing one or upgrade.
X
On Team and Enterprise plans, only Owners can add the connector. Members will not see Meta in the connector list until an Owner adds it at the org level.
X
Custom connectors are still in beta. Tool behavior, OAuth scopes, and permissions can change. Re-check the doc before relying on the MCP for anything mission-critical.
X
"Allow always" hands the agent autonomous access. Per-action approval is the default safety. If you click Allow always for a tool, that tool will fire without asking. Read each scope first.

FAQ

Will using the Meta MCP server get my ad account banned?
No. The MCP server is the official Meta-built integration. The accounts that were getting flagged before this were running janky scripts against the Marketing API the wrong way. The MCP is the sanctioned path.
What is the difference between the MCP and the CLI?
The MCP is a remote connector you paste into Claude. Zero code. Best for chatting with your account. The CLI is a Python tool you install in your terminal. Best for scripts, cron jobs, and CI/CD. The MCP also exposes a few endpoints the CLI does not, like Signal Diagnostics and Help Center search.
Do I need a developer app or app review to use the Meta MCP?
No. The MCP uses Business OAuth or a token link. No developer app required. The CLI may need a developer app for system-token automation. The raw Marketing API requires a full developer app and tokens.
Can I use the Meta MCP on the free Claude plan?
Yes, but the free plan caps you at one custom connector total. If you have used your one slot, the Meta MCP will not fit until you remove the existing one or upgrade.
Can the Meta MCP send Conversions API events or upload offline conversions?
No. CAPI server events, offline conversions, and webhooks are not in the MCP or CLI tool sets. Use the raw Marketing API for those.
Can I use the Meta MCP for local business targeting (city, ZIP, radius)?
Verify before you trust it. The current public CLI docs show country-level targeting for ad set creation, and the MCP is similar. The underlying Marketing API supports city, ZIP, and radius targeting. Draft paused with the MCP, then patch targeting in Ads Manager or use the raw API before going live.
