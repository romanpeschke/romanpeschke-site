Build a Website in Two Minutes

Design a full website with AI, pull it into your IDE, and deploy it live. All free. No code required.

Tools you'll use
Google Stitch Antigravity Vercel Dribbble

Most people building websites with AI still spend way too long getting from idea to live URL. They fight with prompts, copy-paste code between tools, and end up with something that looks generated. This workflow chains three free tools together so you go from a design screenshot to a deployed website in minutes, with code that actually looks like a developer wrote it.

Design
1

Find your visual reference

Go to Dribbble and search for a design style that fits your project. Find something you like, screenshot it. This becomes the creative direction for your AI-generated site.

2

Generate the UI in Google Stitch

  • Open stitch.withgoogle.com and click the "+" button
  • Upload the screenshot you just took
  • Select Web as the design type
  • Write a prompt describing what you want (e.g., "Make me a landing page following this visual style for my coffee business")
  • Select the 3.1 Pro model and hit generate
Connect the tools
3

Get your Stitch API key

  • In Google Stitch, click your profile icon
  • Go to Stitch Settings
  • Scroll to the API Key section
  • Click Create Key and copy it
4

Add the Stitch MCP server to Antigravity

  • Open Antigravity (Google's AI IDE)
  • Go to Settings > Customizations
  • Under "Add MCP Servers", search for Stitch MCP
  • Click + Add to install it
What is MCP?

Model Context Protocol lets your AI IDE talk directly to other tools. In this case, it connects Antigravity to your Stitch designs so the AI can read and convert them into code automatically.

Build and deploy
5

Pull the design into your workspace

  • In Antigravity, create a new workspace and open a fresh folder
  • Tell the agent: "I want to use the Google Stitch MCP server to pull in a website I built"
  • Paste your API key when it asks
  • Point it to the right Stitch project

The agent uses MCP to retrieve your screens, extract the HTML/Tailwind CSS code, and set up the project files locally.

6

Test it locally

The agent builds the project and spins up a local dev server. Click the localhost link it gives you to preview the fully coded, functional website in your browser.

7

Deploy to Vercel

  • Tell the agent: "Looks good, deploy it to Vercel"
  • It packages and deploys the project automatically
  • You get a live .vercel.app URL in seconds

Add a custom domain later in Vercel's domain settings if you want.

X
Don't skip the screenshot step. Stitch generates way better output when it has a visual reference. A text-only prompt gives you generic Bootstrap-looking pages.
X
Don't skip the API key step. The MCP connection is what makes this workflow fast. Without it, you're manually exporting and importing code between Stitch and your IDE.
X
Don't use the free Stitch model. The 3.1 Pro model is significantly better at following your visual reference. The free tier model works, but the output needs more manual cleanup.
Start here

Open Dribbble right now, search "landing page dark" or whatever style fits your niche, and screenshot the first design that catches your eye. Then paste it into Stitch. You'll have a working prototype in under 60 seconds. That's the proof of concept that sells the whole workflow.
